## 1. 기획 개요

### 1.1 목적

삼성전자 채용담당자가 다양한 채널에서 유입되는 후보자 정보를 체계적으로 축적하고, 이력서 기반 역량 데이터를 자동 구조화하며, AI 기반 검색과 필터링을 통해 적합 인재를 빠르게 발굴할 수 있는 인재 Pool 관리 시스템을 구축한다.

본 시스템은 단순 후보자 목록 관리가 아니라, 인재 등록부터 이력서 파싱, 역량 태깅, AI 검색, 후보자 상세 이력 관리, 채용 진행 현황 추적, 재접촉 대상 추천까지 이어지는 채용 운영 플랫폼을 지향한다.

### 1.2 배경

삼성전자 채용 조직은 신입, 경력, 산학, 글로벌, 핵심 기술 인재 등 다양한 인재군을 지속적으로 발굴해야 한다. 그러나 후보자 정보가 엑셀, 이메일, 채용 플랫폼, 추천 메모, 이전 지원 기록 등으로 분산되어 있으면 다음과 같은 문제가 발생한다.

- 우수 후보자 재활용이 어렵고 채용 기회 손실이 발생한다.
- 채용담당자별 관리 기준이 달라 후보자 품질 판단이 일관되지 않다.
- 이력서 검토, 역량 키워드 추출, 후보자 비교에 많은 시간이 소요된다.
- 개인정보 보관 기간, 접근 권한, 활용 동의 이력 관리가 어려워진다.
- 특정 직무나 조직에 적합한 후보자를 빠르게 찾기 어렵다.

### 1.3 제품 비전

"분산된 후보자 데이터를 하나의 신뢰 가능한 인재 자산으로 통합하고, AI를 통해 채용담당자의 탐색, 판단, 후속 액션을 빠르고 정확하게 돕는 시스템"을 만든다.

| 채용 리더 | 채용팀장, HR 리더 | Pool 현황, 직무별 후보자 분포, 채용 파이프라인 가시화 |

| 개인정보 담당자 | 법무/컴플라이언스 | 활용 동의, 보관 기간, 삭제 요청, 접근 이력 관리 |

## 3. 목표 및 비목표

### 3.1 핵심 목표

- 후보자 정보를 중앙화하여 직무, 조직, 기술역량, 경력 수준 기준으로 체계적으로 관리한다.
- 이력서 파일을 업로드하면 주요 경력, 학력, 기술스택, 프로젝트, 자격, 언어, 논문/특허 등을 자동 추출한다.
- AI 기반 자연어 검색으로 "반도체 공정 AI 경험이 있고 글로벌 프로젝트 경험이 있는 5년 이상 경력자" 같은 질의를 지원한다.
- 채용담당자가 후보자별 접촉 이력, 지원 이력, 평가 메모, 상태 변경 내역을 한 화면에서 확인한다.
- 개인정보 보호, 접근 권한, 감사 로그, 활용 동의 상태를 시스템 전반에 내장한다.
- 대시보드에서 인재 Pool의 규모, 품질, 직무별 분포, 업데이트 필요 후보, 채용 전환 지표를 확인한다.

### 3.2 비목표

- 최종 채용 의사결정을 AI가 자동으로 수행하지 않는다.
- 면접 평가 전체 프로세스를 대체하지 않는다.
- 외부 채용 플랫폼의 모든 기능을 복제하지 않는다.
- 개인정보 활용 동의가 없는 후보자의 장기 보관 또는 자동 추천을 허용하지 않는다.
- 특정 성별, 연령, 학교, 출신지 등 민감하거나 차별 소지가 있는 속성 기반 추천을 목표로 하지 않는다.

## 4. 주요 사용 시나리오

### 4.1 후보자 신규 등록

1. 채용담당자가 이력서 파일 또는 후보자 기본 정보를 등록한다.
2. 시스템이 이력서에서 이름, 연락처, 경력, 학력, 기술, 프로젝트, 자격 등 핵심 항목을 추출한다.
3. AI가 직무 관련 키워드, 대표 역량, 경력 요약, 추천 태그를 생성한다.
4. 채용담당자가 추출 결과를 검토하고 수정한 뒤 Pool에 저장한다.

### 4.2 AI 기반 인재 검색

1. 채용담당자가 자연어로 필요한 후보자 조건을 입력한다.
2. 시스템이 조건을 직무, 기술, 경력, 산업, 지역, 언어, 채용 단계 등의 검색 필터로 변환한다.
3. 후보자 목록을 적합도, 최신성, 접촉 가능성, 관심 직무 기준으로 정렬한다.
4. 채용담당자는 추천 근거를 확인하고 후보자를 Shortlist에 추가한다.

### 4.3 프로필 상세 조회 및 이력 관리

1. 채용담당자가 후보자 상세 화면에 진입한다.
2. 기본 정보, AI 경력 요약, 원본 이력서, 기술 태그, 채용 이력, 접촉 이력, 평가 메모를 확인한다.
3. 후보자 상태를 "관심", "접촉 예정", "접촉 완료", "지원 유도", "전형 진행", "보류", "비활성" 등으로 변경한다.
4. 모든 변경 사항은 담당자, 변경 시간, 변경 사유와 함께 이력으로 남는다.

### 4.4 대시보드 모니터링

1. 채용 리더가 전체 인재 Pool 현황을 확인한다.
2. 직무별 후보자 수, 경력 구간, 기술 키워드 분포, 최근 업데이트 후보, 동의 만료 예정 후보를 파악한다.
3. 채용 캠페인 또는 사업부별 전환 지표를 비교한다.
4. 데이터 품질이 낮거나 장기간 업데이트되지 않은 후보자를 정비 대상으로 지정한다.

## 7. AI 기능 기획

### 7.1 AI 활용 영역

- 이력서 텍스트 추출 및 구조화
- 후보자 경력 요약 생성
- 기술/직무/산업 태그 추천
- 자연어 검색어를 구조화 필터로 변환
- 검색 결과 적합도 산정 및 추천 근거 생성
- 유사 후보자 추천
- 후보자 데이터 품질 점검

### 7.2 AI 원칙

- AI는 후보자 탐색과 요약을 보조하며 최종 판단은 사람이 수행한다.
- 추천 결과에는 근거를 제공해 채용담당자가 검증할 수 있어야 한다.
- 민감 정보 또는 차별 가능성이 있는 속성을 추천 기준으로 사용하지 않는다.
- AI 결과는 수정 가능해야 하며, 수정 이력은 관리한다.
- AI 검색 질의와 추천 결과는 감사 및 품질 개선 목적으로 기록한다.

### 7.3 검색 예시

- "NAND 공정 개발 경험이 있고 Python 데이터 분석 역량이 있는 3~7년차 후보자"
- "온디바이스 AI 또는 임베디드 ML 프로젝트 경험이 있는 SW 엔지니어"
- "글로벌 고객 대응 경험이 있는 반도체 품질 엔지니어"
- "석박사급 AI 연구 경험과 논문 실적이 있는 후보자"
- "최근 6개월 내 업데이트된 클라우드/보안 경력자 중 접촉 가능한 후보자"

## 8. 비기능 요구사항

| 영역 | 요구사항 |
|------|----------|
| 성능 | 일반 검색 2초 이내, AI 검색 5초 이내 1차 결과 반환 |
| 확장성 | 후보자 수 수십만 명 이상, 이력서 파일 수백만 건까지 확장 가능해야 함 |
| 보안 | SSO, MFA, RBAC, 데이터 암호화, 다운로드 제한, 감사 로그 필수 |
| 개인정보 | 개인정보보호법 및 내부 보안 정책을 고려한 동의, 보존, 삭제 관리 |
| 가용성 | 업무시간 기준 안정적인 사용을 보장하고 장애 시 복구 절차 제공 |
| 추적성 | 후보자 데이터 변경, AI 처리, 파일 접근, 다운로드 이력 추적 |
| 사용성 | 채용담당자가 후보자 등록부터 Shortlist 생성까지 5분 이내 수행 가능 |
| 품질 | 파싱 결과 신뢰도, AI 추천 근거, 데이터 중복률을 지속 모니터링 |

## 9. 성공 기준

- 이력서 단건 등록 후 후보자 프로필 생성까지 평균 1분 이내 완료된다.
- 이력서 파싱 핵심 필드 정확도가 내부 검수 기준 90% 이상을 달성한다.
- 채용담당자가 자연어 검색으로 원하는 후보자 Shortlist를 5분 이내 생성할 수 있다.
- 중복 후보 등록률이 기존 대비 70% 이상 감소한다.
- 후보자 상세 화면에서 이력서, 접촉 이력, 지원 이력, 평가 메모를 한 화면에서 확인할 수 있다.
- 개인정보 동의 만료 예정 후보를 자동으로 식별하고 담당자에게 알림을 제공한다.
- 모든 후보자 조회, 수정, 다운로드, AI 검색 활동이 감사 로그로 기록된다.
- 파일럿 사용 채용담당자의 만족도 평균 4.2/5.0 이상을 달성한다.
